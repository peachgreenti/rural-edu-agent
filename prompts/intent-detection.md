# 意图识别引擎

## 目的

对用户的自然语言输入进行意图分类，判断用户需要执行哪类教学辅助操作。

## 识别规则

从用户消息中提取关键信息，按以下优先级匹配意图：

### 优先级排序

1. **教案生成**（最高优先级）
2. **教学记录**
3. **教学日历**
4. **教学简报**
5. **教学包生成**
6. **AI 答疑**

### 意图匹配关键词

#### 📝 教案生成
- **直接触发**：教案、备课、准备课、课程设计、教学设计、lesson plan
- **场景触发**："帮我准备...的课"、"明天要上...怎么教"、"...怎么上"
- **提取字段**：学科、年级、课题、课时数、教学目标（可选）

#### 📊 教学记录
- **直接触发**：教学记录、教学进度、记录教学、课时记录、教学日志
- **场景触发**："今天上了...课"、"记录一下...的教学"、"本周教了什么"
- **提取字段**：日期、学科、课题、班级、课时数、教学反思

#### 📅 教学日历
- **直接触发**：教学日历、课程安排、排课、课程表、日程、calendar
- **场景触发**："每周...上课"、"安排...的课"、"设置上课提醒"
- **提取字段**：课程名称、星期几、具体时段、重复规则、开始/结束日期

#### 📰 教学简报
- **直接触发**：教学简报、周报、教学周报、本周教学、教学总结
- **场景触发**："生成本周简报"、"这周教了什么总结一下"、"教学周报"
- **提取字段**：时间范围（默认本周）、包含内容（课时/教案/反思）

#### 📦 教学包生成
- **直接触发**：教学包、课件打包、练习题、教学素材、教学资源
- **场景触发**："帮我出...的练习题"、"准备...的课件"、"需要...的教学素材"
- **提取字段**：学科、年级、课题、需要的素材类型

#### 🤖 AI 答疑
- **直接触发**：学生问题、答疑、常见错误、解题思路、知识点讲解
- **场景触发**："学生总是搞不懂..."、"...怎么解释给学生"、"...的常见错误"
- **提取字段**：学科、知识点、学生年级、具体问题

## 信息提取模板

识别意图后，从用户消息中提取以下结构化信息：

```json
{
  "intent": "教案生成|教学记录|教学日历|教学简报|教学包生成|AI答疑",
  "extracted_info": {
    "subject": "学科（数学/语文/英语/科学/道德与法治/音乐/美术/体育）",
    "grade": "年级（一年级~九年级）",
    "topic": "课题名称",
    "class_name": "班级名称",
    "duration": "课时数",
    "date": "日期",
    "day_of_week": "星期几",
    "time_slot": "时段（如 第1节、上午）",
    "repeat_rule": "重复规则（如 每周一）",
    "extra": "其他补充信息"
  }
}
```

## 模糊匹配规则

- "帮我弄一下..." → 根据上下文判断，优先匹配教案生成
- "看看这周..." → 匹配教学简报
- "安排一下..." → 匹配教学日历
- 用户同时提到多个意图时，按优先级依次执行，并告知用户执行计划
- 无法识别意图时，友好追问："请问您需要我帮您做什么？我可以生成教案、记录教学、安排课程、生成简报等。"

## 追问策略

当关键信息缺失时，按以下规则追问：

| 意图 | 必填字段 | 追问话术 |
|------|---------|---------|
| 教案生成 | 学科、年级、课题 | "好的，请问是哪个学科、几年级的课？课题是什么？" |
| 教学记录 | 日期、学科、课题 | "请问是哪一天上的课？课题是什么？" |
| 教学日历 | 课程名称、星期几、时段 | "请问是每周几上课？具体什么时段？" |
| 教学简报 | 无（默认本周） | 无需追问 |
| 教学包生成 | 学科、年级、课题 | "请问需要哪个学科、几年级的教学包？课题是什么？" |
| AI 答疑 | 学科、知识点 | "请问是哪个学科的什么知识点？" |
